Instructions

This exercise is short, but it requires you to think a bit more carefully about how to implement the requirements. This exercise builds on the 1 star exercise, so begin by copying your code from that exercise into the sketch file for this exercise.

Add a new subclass of Bird called Parrot. Some parrots mimic sounds they hear. The Parrot class should store a collection of phrases that the parrot can say. When a parrot is first created, it should know one phrase, "Squawk!". You will write code that enables a parrot to learn more phrases. To implement this, the Parrot needs a new method, mimic(), which takes one argument, an animal. When this method is called, add the animal's call to the parrot's collection of phrases.

Override the call() method in Parrot. Instead of returning the same phrase each time, randomly select one of its learned phrases to return. For example, if a parrot has learned to mimic a cat and a dog named Spot, its call() method will randomly return one of the following: "Squawk!", "meow", "Hello, my name is Spot".

There is no check file or expected output for this exercise, so you will need to work out how to test it yourself.

The first step to implementing the expected functionality, is to decide what sort of collection you will use to store the learned phrases. When the mimic() method is called, you can use the call() method of the mimic() method argument to "learn" the animal's call by adding it to the collection.

Selecting a random phrase to return for the Parrot's call() method will be easier if you have stored the phrases as an array of strings. In that case, you can generate a random integer corresponding to an index in the array. If you're not sure how to do this, there has been an example already in the module. However, your should try to figure it out on your own first. You have all the knowledge you need!

You should test your methods by creating some example animals (of all child classes), including a parrot or two, and calling the mimic() and call() methods. Due to the random nature of the overridden call() method, you will need to call it multiple times to make sure it is choosing different phrases.
